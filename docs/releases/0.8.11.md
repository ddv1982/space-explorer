# Release 0.8.11 — Helper resurrection-state fix

## Highlights

- Fixes an invalid helper-wing progression state where depleted helper support could be re-granted again on last life.
- Keeps cross-level helper persistence and transition safety behavior from previous releases.

## What changed

- Persistent state model update:
  - `src/systems/PlayerState.ts`
  - Added `helperWing.grantedSlots` to track how many helper slots were granted during the run.
  - Maintains backward compatibility by deriving `grantedSlots` from older slot-only saves when absent.

- Helper grant/runtime update:
  - `src/systems/LastLifeHelperWing.ts`
  - Grant flow now assigns only the next ungranted slot instead of reusing first empty/depleted slot.
  - Captures helper state for granted slots (including spent slots) so depleted slots remain spent.

- Scene persistence call-site compatibility:
  - `src/scenes/GameScene.ts`
  - Updated default helper persistence payload to include `grantedSlots`.

## Gameplay result

- If a helper slot has been fully depleted, it no longer resurrects later in the run.
- You can still progress into an additional support slot when it has not been granted yet.
