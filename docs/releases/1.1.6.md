# Release 1.1.6 — Runtime decomposition and lifecycle hardening

## Highlights

- Keeps the `1.1.x` gameplay line intact while packaging a larger internal decomposition pass as a release-ready patch.
- Breaks scene, system, and music internals into smaller modules so future iteration is safer without changing authored gameplay behavior.
- Hardens lifecycle ownership around runtime teardown, intermission input cleanup, and generated parallax textures.
- Updates the README structure docs so the documented architecture matches the code now on disk.

## What changed

- Scene and flow decomposition:
  - `src/scenes/GameScene.ts`
  - `src/scenes/gameScene/*`
  - `src/scenes/PlanetIntermissionScene.ts`
  - `src/scenes/planetIntermission/interactionController.ts`
  - Split large scene responsibilities into focused runtime/create/update/lifecycle helpers and isolated intermission interaction handling without changing scene sequencing.

- Level music/config organization:
  - `src/config/levels/types.ts`
  - `src/config/levels/music/types.ts`
  - `src/config/levels/music/patterns.ts`
  - `src/config/levels/music/patterns/{arpeggiatorPatterns,bassPatterns}.ts`
  - Reorganized procedural music config/types and pattern data into dedicated modules while removing the stale `musicHelpers.ts` path.

- System helper extraction and lifecycle hardening:
  - `src/systems/audio/procedural/*`
  - `src/systems/effects/*`
  - `src/systems/hud/*`
  - `src/systems/parallax/*`
  - `src/systems/wave/*`
  - `src/scenes/gameScene/runtimeLifecycle.ts`
  - Added clearer ownership boundaries for helper modules and made teardown/cleanup behavior explicit and idempotent where repeated lifecycle paths could otherwise drift.

- Regression coverage and docs sync:
  - `tests/gameSceneRuntimeLifecycle.test.ts`
  - `tests/PlanetIntermissionInteractionController.test.ts`
  - `tests/foregroundSilhouetteLifecycle.test.ts`
  - `tests/levelVisualLayerLifecycle.test.ts`
  - `README.md`
  - Added focused tests for teardown/listener/texture ownership behavior and refreshed the public project-structure docs.
