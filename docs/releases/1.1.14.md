# Release 1.1.14 — Revert viewport policy changes, keep pause-resize fix

## Highlights

- Reverts the unwanted phone-only gameplay viewport consistency changes so desktop and mobile presentation return to the prior behavior.
- Keeps the desktop maximize/resizing bugfix that prevents pause save-slot text from leaking into gameplay after a relayout.

## What changed

- Viewport policy rollback:
  - `src/scenes/gameScene/viewport.ts`
  - `src/scenes/gameScene/viewportScaleRegistration.ts`
  - `src/utils/device.ts`
  - `src/utils/layout.ts`
  - Restored the original full-screen viewport behavior and removed the phone-sized gameplay-field override.

- Gameplay-bound consumer rollback:
  - `src/entities/enemies/Boss.ts`
  - `src/scenes/gameScene/combatFeedbackHandlers.ts`
  - `src/systems/WaveManager.ts`
  - `src/systems/wave/WaveAsteroidSpawner.ts`
  - `src/systems/ParallaxBackground.ts`
  - Reverted dependent gameplay systems back to the original viewport-bound behavior.

- Removed temporary viewport tuning/config scaffolding:
  - `src/config/deviceConfig.ts`
  - `tests/device.test.ts`
  - `tests/gameSceneViewport.test.ts`
  - Removed the phone-screen tuning config and the tests that only existed to support the reverted viewport policy.

- Pause resize regression fix retained:
  - `src/scenes/gameScene/PauseOverlay.ts`
  - `tests/PauseOverlay.test.ts`
  - Kept the relayout fix that reapplies the hidden/visible pause-overlay state so maximize/resizing no longer leaves save-slot text floating in gameplay.
