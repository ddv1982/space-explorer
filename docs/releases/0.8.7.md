# Release 0.8.7 — Boss readability + phase telegraphing

## Highlights

- This release improves boss-fight readability and feel by making phase transitions clearer and more authored.
- Focus areas: phase-change telegraphs, transition pacing, and cleaner per-boss cadence identity.

## What changed

- Runtime telegraph support:
  - `src/systems/GameplayFlow.ts`
  - `src/entities/enemies/Boss.ts`
  - `src/systems/HUD.ts`
  - `src/scenes/GameScene.ts`
  - Added a dedicated boss phase-change event and HUD phase announcement handling.
  - Bosses now support a configurable transition pause before phase-two fire resumes.

- Config extension:
  - `src/config/levels/types.ts`
  - Added optional `phaseTransitionPauseMs` on `BossConfig`.

- Boss readability tuning (Levels 4-10):
  - `src/config/levels/definitions/warzoneCorridor.ts`
  - `src/config/levels/definitions/deepSpaceInferno.ts`
  - `src/config/levels/definitions/wreckfieldRun.ts`
  - `src/config/levels/definitions/ghostlightVeil.ts`
  - `src/config/levels/definitions/crownOfRings.ts`
  - `src/config/levels/definitions/obsidianMaw.ts`
  - `src/config/levels/definitions/terminusBlack.ts`
  - Updated per-boss cooldown, movement, spread/spiral, and transition-pause tuning.

- Attack pattern parameterization:
  - `src/entities/enemies/boss/attacks.ts`
  - `pursuit` and `bulwark` styles now scale more directly from configured spread/shot values.
