# Release 1.1.8 — Pause controls, checkpoints, and music UI polish

## Highlights

- Adds a full-featured in-game pause surface with checkpoint slot controls and live music tuning.
- Introduces persistent local save slots shared between the main menu and pause flow.
- Unifies the new neon UI treatment across pause/menu controls and tightens responsive spacing on compact and desktop layouts.

## What changed

- Pause overlay + control flow:
  - `src/scenes/gameScene/PauseOverlay.ts`
  - `src/scenes/gameScene/PauseStateController.ts`
  - `src/scenes/gameScene/pauseOverlay/{controls,types,view}.ts`
  - `src/scenes/gameScene/pauseViewportWiring.ts`
  - Added resume/save/load/delete/main-menu actions, pause status messaging, orientation-lock handling, responsive slot-row layout, and desktop/compact spacing polish.

- Menu save-slot + music-lab upgrade:
  - `src/scenes/MenuScene.ts`
  - `src/scenes/menuScene/{layout,panels}.ts`
  - Added loadable/deletable save tiles and aligned the main-menu music lab with the shared runtime tuning controls.

- Shared UI extraction:
  - `src/scenes/shared/{actionButtonControl,musicSliderCluster,musicSliderControl,musicSliderIcons,neonUiTheme,textFit}.ts`
  - Centralized the new button, slider, icon, frame, and text-fitting behavior used by pause and menu interfaces.

- Checkpoint persistence + player-state normalization:
  - `src/systems/SaveSlotStorage.ts`
  - `src/systems/PlayerState.ts`
  - `src/entities/Player.ts`
  - Added versioned localStorage-backed save slots and hardened recovery/normalization for legacy or malformed run-state payloads.

- Audio runtime tuning:
  - `src/systems/audio/ProceduralMusicManager.ts`
  - `src/systems/audio/procedural/runtimeControl.ts`
  - `src/scenes/shared/musicRuntimeTuning.ts`
  - `src/scenes/shared/musicSliderCluster.ts`
  - Wired volume/tuning controls through the shared audio path and raised the runtime output floor so gameplay/pause loudness stays aligned with the main menu baseline.
