# Release 1.1.13 — Phone-only gameplay consistency and pause-resize fix

## Highlights

- Keeps the gameplay-consistency viewport fix targeted to actual phone-sized touch screens instead of changing desktop and larger-resolution behavior.
- Fixes the desktop maximize/resizing glitch where pause save-slot text could remain visible in gameplay after a relayout.
- Makes the phone-screen cutoff tunable from one config file for faster future adjustments.

## What changed

- Phone-only gameplay viewport policy:
  - `src/utils/device.ts`
  - `src/utils/layout.ts`
  - `src/scenes/gameScene/viewport.ts`
  - `src/scenes/gameScene/viewportScaleRegistration.ts`
  - Narrowed the fixed gameplay-field behavior to coarse-touch phone-sized viewports only, while desktop and larger resolutions keep the original full-screen gameplay behavior.

- Gameplay systems now follow the active viewport policy:
  - `src/entities/enemies/Boss.ts`
  - `src/scenes/gameScene/combatFeedbackHandlers.ts`
  - `src/systems/WaveManager.ts`
  - `src/systems/wave/WaveAsteroidSpawner.ts`
  - `src/systems/ParallaxBackground.ts`
  - Updated gameplay-bound consumers so they use the active gameplay bounds instead of always forcing the phone-sized logical field.

- Tunable phone cutoff config:
  - `src/config/deviceConfig.ts`
  - Centralized the phone short-side/long-side viewport thresholds so device tuning no longer requires editing detection logic directly.

- Pause resize regression fix:
  - `src/scenes/gameScene/PauseOverlay.ts`
  - Ensured relayout reapplies the current hidden/visible overlay state so desktop maximize/resizing no longer leaves save-slot text floating in gameplay.

- Regression coverage:
  - `tests/device.test.ts`
  - `tests/gameSceneViewport.test.ts`
  - `tests/PauseOverlay.test.ts`
  - plus updated supporting viewport/device mocks in related tests.
