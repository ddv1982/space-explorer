# Release 1.1.9 — Mobile menu spacing and touch pause follow-up

## Highlights

- Fixes the mobile main-menu subtitle/logo overlap seen on small iPhone-class screens.
- Adds a dedicated touch-only pause button in the top-right gameplay HUD area for mobile players.
- Keeps touch pause taps from accidentally triggering weapon fire on the right half of the screen.

## What changed

- Mobile menu title-band polish:
  - `src/scenes/menuScene/layout.ts`
  - `src/scenes/menuScene/panels.ts`
  - Rebalanced very-short compact title/subtitle spacing, reduced mobile title size/ornament span, and enabled wrapped subtitle layout so the session prompt no longer intersects the logo.

- Mobile pause affordance:
  - `src/systems/MobileControls.ts`
  - `src/scenes/gameScene/runGameSceneCreateBootstrap.ts`
  - Added a touch-only top-right pause button, relayout handling, blocked-state visibility, and pause-controller wiring for mobile gameplay.

- Touch-input filtering:
  - `src/systems/InputManager.ts`
  - Excluded control-reserved touch pointers from mobile fire detection so the pause button does not behave like a shoot tap.
