# Release 1.1.24 — teach the campaign to read shelter lanes

Open the first vertical slice of the gameplay-diversity redesign: authored pressure beats, deliberate Recovery Beat relief, and literal cover that turns wreckage into readable shelter instead of scenery.

## Highlights

- Adds authored section primitives for lane-based signature waves and health/shield Recovery Beat drops.
- Adds literal cover: selected asteroid terrain blocks enemy bullets and bomber bombs, while enemy bodies remain dangerous.
- Reworks Level 6 into a terrain-as-cover route with breakable salvage plates and a bulwark-style boss pressure profile.
- Adds vertical-slice content for Level 1 lane-reading, Level 5 endurance relief, and Level 10 adaptive synthesis waves.
- Adds warning/error guardrails for authored waves, recovery drops, mobile lane-density risk, and cover durability mistakes.
- Captures the new campaign language, ADR, implementation plan, and manual playtest checklist.

## Lore

Constraint: Preserve the existing weighted encounter system, player economy, and early onboarding difficulty while adding authored beats only where the approved slice needs predictable structure.
Rejected: Global power-up rebalance | Recovery needed reliable health/shield relief in authored spots, not a campaign-wide random-drop economy change.
Rejected: New cover game-object hierarchy | Asteroids already carried durability, collision damage, and indestructibility metadata, so cover belongs as opt-in asteroid behavior.
Rejected: Exact-pixel signature waves | Lane templates keep authored moments readable across desktop and narrow viewports.
Confidence: high
Scope-risk: medium
Directive: Let the campaign teach players to read lanes, relief, cover, and synthesis through authored moments while keeping random pressure and enemy-body danger intact.
Tested: `bun test tests/WaveManager.test.ts`; `bun test tests/CollisionManager.test.ts tests/WaveAsteroidSpawner.test.ts`; `bun test tests/levelCampaigns.test.ts tests/WaveManager.test.ts tests/CollisionManager.test.ts`; `bun test`; `bun run levels:validate`; `bun run lint`; `bun run build`; `bun run bundle:check`; `bun run knip`.
Not-tested: Manual desktop/narrow-viewport playtest is still required using `docs/plans/gameplay-diversity-redesign-2026-06-13.md`; non-slice levels 2, 3, 4, 7, 8, and 9 remain planned but not implemented in this release.
