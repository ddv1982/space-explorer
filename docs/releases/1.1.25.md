# Release 1.1.25 — harden the quality corridor

Turn the quality improvement plan into release-ready guardrails: repeatable CI checks, safer pooled visuals, leaner parallax lifecycle wiring, and campaign validation that speaks the same language as the level design notes.

## Highlights

- Adds Bun-based CI for typecheck, lint, tests, level validation, Knip, build, and bundle guardrails.
- Adds `bun run typecheck` with dedicated test TypeScript coverage while keeping production TypeScript strict.
- Removes per-frame gameplay behavior allocation from `GameScene.update` by caching the frame behavior after scene bootstrap.
- Guards delayed tint callbacks on pooled or respawning enemies, bosses, asteroids, and helper ships with lifecycle tokens.
- Deletes the empty foreground silhouette lifecycle seam from the parallax stack.
- Adds warning-level campaign validation for Within-Level Pacing, Dominant Motif, Lane-Reading, Ambush Anticipation, and Recovery Beat authoring.
- Refreshes bundle/startup notes with current Vite 8 lazy-scene metrics and documents manual release smoke coverage.

## Lore

Constraint: Preserve the current Phaser/Vite/Bun stack, GameScene orchestration ownership, pooled entity architecture, and authored campaign behavior while turning the quality plan into repeatable release gates.
Rejected: Full strict typing for every Phaser test stub | The first test type-check gate needs to catch import and contract drift without forcing a broad fake-Phaser framework into this release.
Rejected: General-purpose pooling framework | Local lifecycle tokens close the stale delayed-callback risk without adding a new abstraction around every pooled object.
Rejected: Keep dormant foreground silhouettes | The seam was no longer a near-term art feature, so deleting the no-op path is clearer than preserving inert lifecycle wiring.
Confidence: high
Scope-risk: medium
Directive: Keep the quality corridor readable: local gates match CI, pooled visuals cannot bleed across lives, and campaign validation protects authored pressure language before playtest.
Tested: `bun run build`; `bun run typecheck`; `bun run lint`; `bun run test`; `bun run levels:validate`; `bun run knip`; `bun run bundle:check`.
Not-tested: Manual browser/mobile smoke was not run in this terminal session; the release notes keep the manual smoke checklist as the required follow-up for scene flow, mobile controls, and audio unlock behavior.
