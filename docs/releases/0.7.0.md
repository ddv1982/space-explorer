# Release 0.7.0 — Deterministic procedural music architecture

## Highlights

- This release delivers a full procedural music architecture upgrade across composition, runtime scheduling, testing, and authoring workflow.
- Music playback now combines explicit intent metadata, meter-aware timing, harmonic progression, polyrhythmic gating, and arrangement arcs in a deterministic pipeline.

## Music system upgrades

- Added strict `MusicIntent` metadata across level stage/boss tracks, including:
  - deterministic seed
  - time signature
  - compositional descriptors (mode, rhythmic feel, energy profile)
- Added first-class harmonic progression modeling (`degree`, `barsDuration`, `quality`, optional octave shift).
- Added meter-aware scheduling support and odd-meter rollout (`7/8`) in campaign content.
- Added deterministic per-layer rhythm scheduling:
  - division/phase/gate
  - accent-driven gain shaping
  - arrangement-driven density/gain modulation
- Added arrangement conductor behavior (`intro`, `build`, `peak`, `release`) with bar-based deterministic resolution and runtime influence.

## QA and tooling

- Expanded level validation with explicit stage/boss music contract checks for:
  - track core fields
  - intent/time-signature validity
  - harmony progression constraints
  - arrangement section constraints
  - per-layer rhythm bounds
- Added deterministic replay fixtures that lock combined resolver timelines (meter + harmony + arrangement + rhythm) for both `4/4` and `7/8` scenarios.
- Added and expanded focused tests for meter context, harmony resolution, arrangement behavior, and rhythm modulation boundaries.

## Documentation
