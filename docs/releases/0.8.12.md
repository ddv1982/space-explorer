# Release 0.8.12 — Runtime performance stability pass

## Highlights

- Prevents inactive pooled entities from participating in Arcade physics after despawn.
- Reduces resize-related visual churn by coalescing noisy viewport events before expensive rebuild/restart work.
- Trims hot-path allocation overhead in boss targeting and flash/effects callback flows.

## What changed

- Pooled entity lifecycle hardening:
  - `src/utils/entityUtils.ts`
  - `src/entities/Bullet.ts`
  - `src/entities/EnemyBullet.ts`
  - `src/entities/BomberBomb.ts`
  - `src/entities/Asteroid.ts`
  - `src/entities/PowerUp.ts`
  - `src/entities/enemies/EnemyBase.ts`
  - Added shared `spawnEntity` lifecycle helper and updated despawn behavior to disable Arcade body participation while inactive.

- Resize/parallax churn reduction:
  - `src/systems/ParallaxBackground.ts`
  - `src/scenes/shared/registerRestartOnResize.ts`
  - `src/main.ts`
  - Added short debounce guards and rounded-dimension checks to avoid repeated rebuild/restart/refresh work caused by viewport jitter.

- Boss + effects hot-path cleanup:
  - `src/entities/enemies/Boss.ts`
  - `src/scenes/GameScene.ts`
  - `src/systems/EffectsManager.ts`
  - `src/entities/Player.ts`
  - `src/entities/HelperShip.ts`
  - `src/entities/Asteroid.ts`
  - `src/entities/enemies/EnemyBase.ts`
  - Boss now uses an injected/cached player reference with safe fallback lookup; flash/effects timers/callbacks were adjusted to reduce per-event allocation churn.

## Gameplay result

- Fewer ghost/phantom interactions from inactive pooled entities.
- Smoother behavior during rapid viewport/resize changes with less repeated scenic rebuild pressure.
- Lower hot-path callback/allocation overhead in heavy combat moments.
