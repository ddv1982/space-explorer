# Release 0.7.5 — Simplified music flow follow-up

## Highlights

- This follow-up responds to listening feedback that `0.7.4` felt over-tuned.
- The procedural music runtime is simplified back toward the `0.7.3` feel, while keeping only subtle continuity smoothing.

## What changed

- Reverted rhythm threshold logic in `src/systems/audio/procedural/rhythm.ts` to the prior grid-threshold behavior used before `0.7.4`.
- Removed scheduler step-skipping drift recovery in `src/systems/audio/ProceduralMusicManager.ts` to avoid discontinuity from dropped scheduled steps.
- Applied a smaller, low-risk continuity tune:
  - scheduler constants adjusted from `50ms / 0.18s` to `45ms / 0.22s` (polling / ahead window),
  - tone layer duration factor adjusted from `0.92` to `0.96` in `src/systems/audio/procedural/synthesis.ts` for slightly smoother note connection.
- Restored deterministic fixture expectations and cadence assertions in:
  - `tests/musicReplayDeterminism.test.ts`
  - `tests/polyrhythm.test.ts`

## Notes

- No level definition files were changed in this patch.
- Level music authoring logic remains consistent; updates are limited to shared runtime behavior.
