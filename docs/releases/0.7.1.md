# Release 0.7.1 — Music feel stabilization pass

## Highlights

- This patch smooths the procedural music feel after the larger 0.7.0 architecture rollout.
- Focus is on reducing irregular dropouts and sluggish section feel while keeping the deterministic system intact.

## Music feel improvements

- Stabilized rhythm trigger behavior in `rhythm.ts`:
  - replaced hash-like per-pulse dropout behavior with deterministic grid-threshold gating.
  - keeps playback reproducible while producing steadier rhythmic phrasing.
- Rebalanced scheduler modulation in `ProceduralMusicManager`:
  - arrangement density now controls trigger cadence directly.
  - layer gain multipliers now shape loudness only (no compounded sparsity).
- Retuned default stage/boss rhythm and arrangement activity floors in `reimaginedMusic.ts`:
  - raised baseline gate values.
  - raised intro/release section density and energy support.
  - preserved section contrast while improving continuity.

## Deterministic fixtures and validation

- Updated deterministic replay fixture expectations to match stabilized scheduling behavior.
- Validation run:
  - `bun test`
  - `bun run lint`
  - `bun run build`
  - `bun run levels:validate`
