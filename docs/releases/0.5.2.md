# Release 0.5.2 — Adaptive progression and level-authoring safeguards

## Highlights

- Added **adaptive boss durability scaling** based on player upgrade investment and campaign progression.
- Added **hazard fairness pacing controls** to reduce unfair overlap spikes between scripted hazards and regular encounter waves.
- Added **section identity hooks** that shape spawn and music intensity behavior over section progress using tension arcs and VAT tension bias.
- Added a new `bun run levels:validate` command to enforce level-config invariants.

## System changes

### Boss progression scaling

- Boss HP now scales with:
  - current campaign level progress
  - player upgrade pressure (HP / damage / fire-rate / shield)
- Scaling is bounded to preserve readability and avoid runaway fights.

### Hazard pacing fairness

- Hazards now support optional duration windows via `durationMs`.
- Wave spawning uses a decaying hazard-pressure budget to smooth extreme hazard+enemy overlap periods.
- Encounter interval and count are throttled under high hazard pressure.

### Section identity scaffolding

- Added reusable section-identity helpers used by both `GameScene` and `WaveManager`.
- Introduced section-progress-driven shaping for:
  - spawn-rate scaling
  - music-intensity scaling
- Existing levels gain default behavior via phase-based tension-arc fallbacks.

### Level validation command

- Added `scripts/validateLevels.ts` and `levels:validate` script.
- Validator checks include:
  - section ordering and contiguity
  - section coverage against expected terminal progress
  - boss trigger and boss config coherence
  - hazard parameter sanity bounds

## Validation run

- `bun run levels:validate`
- `bun run lint`
- `bun run build`
- `bun run knip`
