# Release 0.8.15 — Bundle optimization patch

## Highlights

- Reduces startup payload by loading heavy gameplay/ending scenes only when they are first needed.
- Splits Phaser and third-party dependencies into explicit build chunks for cleaner app entry delivery.
- Preserves first-use transition safety by registering scenes before scene starts and deduplicating concurrent loads.

## What changed

- Lazy scene registration:
  - `src/main.ts`
  - `src/scenes/sceneRegistry.ts`
  - Startup now includes only `BootScene`, `PreloadScene`, and `MenuScene`; gameplay/ending scenes are dynamically imported and registered on demand.

- First-use scene start safety:
  - `src/scenes/MenuScene.ts`
  - `src/scenes/GameScene.ts`
  - `src/scenes/PlanetIntermissionScene.ts`
  - Replaced direct `scene.start(...)` calls for lazily loaded targets with `startRegisteredScene(...)` to avoid missing-scene starts.

- Regression coverage for async scene loading:
  - `tests/sceneRegistry.test.ts`
  - Added tests for concurrent registration deduplication and load-then-start ordering.

- Vite chunk strategy update:
  - `vite.config.ts`
  - Added manual chunking to emit a dedicated `phaser` chunk (`node_modules/phaser`) plus a separate `vendor` chunk for other dependencies.

## Build impact

- App entry chunk dropped from baseline `dist/assets/index-Dc0mJv79.js` at 1,623.84 kB (gzip 418.53 kB) to `dist/assets/index-0KqbZ6Bn.js` at 34.02 kB (gzip 10.46 kB).
- Largest output chunk moved to dedicated `phaser` chunk (`dist/assets/phaser-CS3aEib-.js`: 1,351.46 kB, gzip 348.50 kB), reducing initial app entry weight while keeping engine code cacheable.
