# Release 0.7.4 — Music flow continuity patch

## Highlights

- This patch targets audible flow continuity so procedural tracks feel more consistently connected and less prone to odd interruption pockets.
- Focus areas are deterministic rhythm trigger distribution and playback scheduler drift recovery.

## What changed

- Updated rhythm scheduling in `src/systems/audio/procedural/rhythm.ts`:
  - replaced clustered threshold gating with a deterministic distributed trigger strategy for sub-unity `gate` and `density` values,
  - kept deterministic replay guarantees while reducing long contiguous mute clusters that can sound like interruptions.
- Improved transport stability in `src/systems/audio/ProceduralMusicManager.ts`:
  - increased schedule-ahead window,
  - tightened scheduler polling,
  - added drift recovery that skips missed steps and re-anchors scheduling to near-future time when timer jitter falls behind.
- Expanded cadence regression coverage in `tests/polyrhythm.test.ts`:
  - added anti-clustering test for sub-unity gate behavior,
  - updated odd-meter cadence assertions to verify deterministic outcomes and bounded rest runs instead of brittle full-sequence snapshots.
- Updated deterministic replay fixtures in `tests/musicReplayDeterminism.test.ts` to reflect the new distributed gate behavior.
