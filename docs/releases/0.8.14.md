# Release 0.8.14 — Respawn telemetry and death-cue burst tuning

## Highlights

- Adds an opt-in respawn frame probe for transition-time diagnostics without changing default gameplay behavior.
- Reduces player-death burst particle load to lower hitch risk on the death-to-respawn boundary.
- Preserves prior single-clock respawn flow and physics-world-only freeze scope from recent lifecycle hardening.

## What changed

- Respawn transition instrumentation:
  - `src/scenes/gameScene/respawnFrameProbe.ts`
  - Added a dedicated probe that can be enabled via `?debugRespawnFrameProbe=1` or `globalThis.__SPACE_EXPLORER_RESPAWN_FRAME_PROBE__ = true`, logging transition duration, frame count, delta stats, and over-budget frame counts.

- Flow integration for probe sampling:
  - `src/scenes/gameScene/GameSceneFlowController.ts`
  - `src/scenes/GameScene.ts`
  - Added probe begin/finish hooks in respawn lifecycle and frame sampling during gameplay-locked update windows.

- Player-death FX budget reduction:
  - `src/systems/EffectsManager.ts`
  - `src/scenes/GameScene.ts`
  - Extended `createExplosion(...)` with optional `particleBudgetScale` and tuned the player death cue to use a lower particle budget while retaining visual identity.

## Gameplay result

- Lower peak burst load on player death transitions with expected smoother frame pacing on constrained devices.
- Better observability for respawn hitch investigation via explicit transition telemetry when debug mode is enabled.
