# Release 1.1.1 — Refactor readiness pass

## Highlights

- Keeps the `1.1.0` gameplay and presentation line intact while packaging the refactor work as a small patch release.
- Breaks large scene, parallax, and sprite-generation modules into focused helpers so future iteration is safer.
- Consolidates generated-texture and scene-event plumbing behind shared utilities instead of repeating setup code.

## What changed

- Gameplay and scene organization:
  - `src/scenes/GameScene.ts`
  - `src/scenes/gameScene/sceneEvents.ts`
  - `src/scenes/gameScene/viewport.ts`
  - `src/scenes/PlanetIntermissionScene.ts`
  - `src/scenes/planetIntermission/presentation.ts`
  - Extracted viewport, scene-event, and intermission-presentation helpers from large scene files without changing scene flow.

- Procedural rendering organization:
  - `src/systems/ParallaxBackground.ts`
  - `src/systems/parallax/backgroundMotion.ts`
  - `src/systems/parallax/atmosphereProfile.ts`
  - `src/systems/parallax/moonSurfaceGenerator.ts`
  - `src/systems/parallax/passingPlanetGenerator.ts`
  - `src/systems/parallax/planetTextureGenerator.ts`
  - `src/systems/parallax/scenicTextureGenerator.ts`
  - `src/systems/parallax/starfieldTextureGenerator.ts`
  - Moved reusable motion and texture-generation behavior into dedicated parallax helpers while preserving existing level visuals.

- Shared texture generation and sprite modules:
  - `src/utils/generatedTexture.ts`
  - `src/utils/SpriteFactory.ts`
  - `src/utils/spriteFactory/*.ts`
  - Replaced the monolithic sprite factory with smaller themed modules and a shared generated-texture wrapper.
