# Release 0.8.13 — Death/respawn hitch patch

## Highlights

- Removes cross-clock respawn timing divergence by eliminating browser-timer watchdog behavior.
- Narrows freeze/resume scope from full scene pause/resume to Arcade physics world pause/resume.
- Preserves existing death/respawn gameplay while reducing transition hitch risk around life-loss recovery.

## What changed

- Respawn timing coherence update:
  - `src/scenes/gameScene/GameSceneFlowController.ts`
  - Removed the respawn watchdog `setTimeout` path and its related cleanup state so respawn completion no longer relies on browser timing while scene time can be paused.

- Pause scope reduction for respawn freeze:
  - `src/scenes/GameScene.ts`
  - Updated flow context pause/resume hooks to pause/resume `physics.world` instead of pausing/resuming the full scene lifecycle.

## Gameplay result

- Smoother-feeling death-to-respawn transitions with less chance of pause/resume hitching.
- More consistent transition timing by keeping respawn control in a single runtime timing model.
