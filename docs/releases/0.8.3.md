# Release 0.8.3 — Level design research + core pacing uplift

## Highlights

- This release focuses on making current levels feel more readable, intentional, and enjoyable through research-backed pacing improvements.
- Work was done in two layers: authoring guardrails (validator) and direct tuning of core campaign levels.

## What changed

- Added level design research synthesis:
  - `docs/level-design-research-notes.md`
  - Captures actionable heuristics from Exa/Ref research on pacing curves, telegraphing, and shmup readability.
- Added pacing/readability guardrails to `scripts/validateLevels.ts`:
  - warns for weak section arc composition,
  - warns for shallow non-boss release windows,
  - warns for abrupt hazard cadence/intensity jumps,
  - warns for too-fast/too-harsh hazard telegraphing in early levels.
- Retuned core campaign level definitions:
  - `src/config/levels/definitions/asteroidBeltAlpha.ts`
  - `src/config/levels/definitions/nebulaPass.ts`
  - `src/config/levels/definitions/ionStormSector.ts`
- Introduced explicit section-level `tensionArc` + `vatTarget` shaping on these levels and adjusted hazard cadence/intensity and spawn pressure to improve climb/release feel.

## Validation

- `bun run levels:validate`
- `bun test`
- `bun run build`
